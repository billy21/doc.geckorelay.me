# Surge 3 Pro 配置

# 概述

Surge 3 Pro 是 Surge 2 for iOS 的升级版，支持了更多新功能，并且购买方式更新为「应用内购买」，并可以导出授权到邮箱，用来避免某个国家或地区的 App Store 下架应用程序导致购买者不能更新。

GeckoRelay 支持 Surge 3 Pro 的托管配置功能，并将很快支持 AEAD 加密算法和 Simple-OBFS 插件。

❗️ 这是一个付费软件，你需要购买才能使用。

主流支持

GeckoRelay 目前提供对此客户端的主流支持

# 检查 GeckoRelay 订阅的设置

WARNING

此步骤非常重要，必须按照此步骤检查无误后才能正常使用 Surge

Surge 3 Pro 并不兼容 SSR 协议，因此需要在 GeckoRelay 管理门户将所有协议组设置为「兼容模式」才能使用。

访问 GeckoRelay 管理门户，点击已经激活的订阅进入订阅管理页面。然后点击「订阅设置」按钮，确保所有插件都如下图所示包含「兼容」字样：

# 获取云端托管型配置

打开 GeckoRelay 管理门户，点击订阅名称进入订阅信息管理页面。在「自动配置」功能区找到Surge – 托管配置。

按下「托管配置」的按钮，按钮应当显示「已复制到剪贴板」。

这个 API URI 非常重要，你应当把它当做密码一样妥善保管，避免泄露。

# 配置 Surge 3 Pro

打开 Surge 3 Pro

点击左上角Default附近的倒三角打开配置管理。然后点击「从 URL 下载配置」。

在弹出的输入框中连点两次屏幕，选择「粘贴」，以将刚才复制到的云端托管型配置 URL 粘贴到窗口中。

完成后，点击「下载」。

现在，配置列表中会出现一个名称为GeckoCloud的配置文件，并显示为托管配置，最后更新于 0 分钟前。

GeckoRelay 的托管型配置每 2 天自动更新一次，如果希望提前更新托管配置，只需要在对应的配置上向左清扫，选择「立即更新」。

点击右上角的「完成」，返回到主界面。

通过点击底部的「策略组」选项卡，将可以在不同的 GeckoRelay 接入点中选择。

选择所需的接入点后，点击「首页」选项卡回到主界面。

将主界面的「出站模式」更改为「规则模式」，然后打开 「Rewrite」的开关。

点击顶部GeckoCloud旁的开关打开 Surge，初次使用时会提示需要下载外部模块。

下载完成外置模块后，需要再次点击顶部GeckoCloud旁的开关打开 Surge，这时会提醒需要向系统添加一个 VPN 连接，请点击 Allow。

系统会跳转到设置，并要求输入密码（使用 Face ID 或无 Touch ID 的设备），或要求验证指纹（有 Touch ID 的设备）。

添加完成后会返回到 Surge 3 Pro 主界面，这时我们还需要再一次点击顶部GeckoCloud旁的开关打开 Surge。

由于 Apple 对使用 Network Extension 的应用程序有内存要求，而 GeckoRelay 的配置文件中有 2000+ 条规则（实际条目远少于 GitHub 上的热门 Surge 规则项目），因此 Surge 3 Pro 会弹出一个提示框，无需理会，点击「好的」即可。
